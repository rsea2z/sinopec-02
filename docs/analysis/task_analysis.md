# SINOPEC-02 任务分析

_面向钻井轨迹关键点自动识别的任务拆解与建模约束说明。_

---

## 任务本质

该课题不是常规的独立样本表格分类，而是 **按井分组的轨迹序列关键点识别**。

从数据结构看，每口井都是一条按 `XJS` 单调递增的测点序列。标签 `关键点` 只标记极少数位置，用来指出轨迹工况切换的起点。结合轨迹形态，可以将标签解释为：

- `1`: 增斜点
- `2`: 稳斜点
- `3`: 降斜点

大多数井的标签顺序满足 `1 -> 2 -> 3`，且 `3` 允许缺失。

## 数据层面结论

- `train.csv` 中混合了两类点：
  - 实际轨迹点：带 `JX/FW/LJCZJS/关键点`
  - 设计轨迹点：带 `JX_design/FW_design/LJCZJS_design`
- `sample_submission.csv` 只覆盖验证集中的实际轨迹点
- 训练井和验证井不重合，必须按井做泛化评估
- 类别极不平衡，关键点数远小于普通点

## 建模挑战

### 1. 极端类不平衡

如果直接做逐点四分类，模型容易被 `0` 类主导。因此：

- 不能只看 accuracy
- 应重点看 macro-F1、关键点召回率
- 需要引入结构化约束或候选点筛选

### 2. 设计轨迹与实际轨迹分离

设计轨迹不是和实际轨迹逐点对齐存储的，因此必须先对齐后才能利用。合理做法是：

- 以 `XJS` 为主轴
- 对 `JX_design`、`LJCZJS_design` 做数值插值
- 对 `FW_design` 做圆周展开后插值

### 3. 方位角是圆周变量

`FW` 取值在 `0-360` 度之间，直接相减会出现跨 `0/360` 边界的问题，因此：

- 需要用圆周差值
- 或转为 `sin/cos` 特征

### 4. 标签是结构化的

同一口井内，标签出现顺序不是任意的。模型输出最好满足：

- `1` 在 `2` 之前
- `2` 在 `3` 之前
- `3` 可以不存在

这意味着后处理非常有价值。

## 推荐方案

### 第一阶段：可靠 baseline

目标不是追求最复杂模型，而是先建立可信基准。

建议流程：

1. 分离实际轨迹点与设计轨迹点
2. 将设计轨迹对齐到实际测点
3. 做特征工程
4. 使用 `GroupKFold` 按井交叉验证
5. 训练点级分类器
6. 再做每井结构化后处理

### 第二阶段：更强模型

在 baseline 之上，可以尝试：

- `BiLSTM/TCN/Transformer` 序列模型
- 变点检测 + 分类联合方案
- 两阶段方法：
  - 先找候选变点
  - 再在候选点中做精细分类

## 研究价值

该题目的价值不只是分类准确率，而是将工程师经验中的轨迹阶段识别规则数据化、模型化。真正有意义的成果通常包括：

- 自动识别关键点
- 给出可解释特征或判据
- 能够迁移到未见井
- 对设计轨迹偏差有工程解释
